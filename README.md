# kor_goemotion Project

**졸업논문 프로젝트: 단순 긍·부정 분석을 넘어 기계가 인간의 복잡한 감정을 정교하게 이해하도록 설계된 고차원적 한국어 감정 인지 모델**

---

## **0. 연구기간**
2023.09 ~ 2023.12

---

## **1. 핵심 요지**
본 연구는 기존 자연어 처리 분야의 감정 모델이 약 6~8개로 제한되어 인간의 다양한 정서를 충분히 이해시키기에 부족하다는 문제점과, 
'감정'과 '감성'의 개념이 혼재되어 사용되는 한계를 극복하기 위해 수행되었습니다. 이를 위해 27가지 고유 감정을 제시한 [GoEmotions](https://github.com/google-research/google-research/tree/master/goemotions) 체계를 기반으로 데이터를 재구조화하고, 
한국어 특화 모델인 **KoBERT**를 활용하여 세분화된 감정 인식의 가능성을 구조적으로 확인하였습니다.

---

## **2. 단계별 설명**

### **2.1 데이터 아키텍처 및 재분류**
* **데이터 소스**: AI Hub의 '한국어 감정 정보가 포함된 단발성 대화 데이터셋'(38,594개 문장)과 '감성 대화 말뭉치'(51,630개 문장 세트)를 활용하였습니다.
* **감정 라벨링**: Alan Cowen의 연구를 종합한 27가지 감정 분류 체계를 채택하여 , 전체 데이터를 '갈망', '비통', '당혹감' 등 세분화된 라벨로 전면 재분류하였습니다.
* **전처리 과정**: 의미 분별이 분명하도록 띄어쓰기, 복합어, 숫자 및 외국어 처리를 수행하고, KoBERTTokenizer를 통해 형태소 단위로 토크나이징 하였습니다.

### **2.2 실험 설계 및 모델링**
* **모델 구조**: SKT Brain에서 공개한 [KoBERT](https://github.com/SKTBrain/KoBERT)를 기반으로, 한국어 특유의 언어 변화 특성을 반영한 사전 학습 모델을 활용하였습니다.
* **하이퍼파라미터**: Batch size 16, Epochs 5(사전학습)/3(성능검증), Learning rate $5 \times 10^{-5}$를 설정하였습니다.
* **학습 스케줄러**: 학습 안정성을 위해 `get_cosine_schedule_with_warmup`을 적용하여 가중치 업데이트를 조절하였습니다.

---

## **3. 주요 실험 결과**
* **학습 추이**: 사전학습 결과 1회차 40.91%에서 5회차 89.13%로 분류 정확도가 크게 향상됨을 확인하였습니다.
* **특이점**: '감사', '배려', '짜증'과 같이 명확한 키워드가 존재하는 감정은 분류 성능($F1-score$)이 높게 나타났습니다.
* **성능 검증**: 장문의 대화 데이터셋에서도 반복 학습에 따라 정확도가 상승하는 경향성을 입증하였습니다.

---

## **4. 예외 및 주의점**
* **정서적 불일치**: 서구권 기반의 GoEmotions 모델이 한국 정서와 완전히 부합하지 않아, '질투'와 같이 한국어 대화에 빈번한 감정이 누락된 한계가 있습니다.
* **분류 난이도**: '반감', '비통', '실망' 등은 한국어 특유의 표현 방식(반어법, 설의법 등)이나 명확한 판단 기준의 모호함으로 인해 상대적으로 성능이 저조하였습니다.
* **기술적 한계**: KoBERT의 제한된 Vocab 사이즈(약 8,000개)로 인해 다양한 문맥적 자질을 충분히 반영하지 못한 측면이 존재합니다.
* **시기적 한계**: 본 연구는 LLM 모델이 비약적으로 발전하기 전인 2023년 11월 시점에 진행되었습니다. 최신 연구는 전혀 반영되어 있지 않습니다.
* **참고사항**: 논문에 명시된 저자명은 개명 전 이름입니다.

## **5. 판단 기준 (성능 측정 지표)**
본 프로젝트는 다중 클래스 분류의 엄밀성을 위해 다음 지표를 활용하였습니다.

* **정밀도(Precision)**: 모델이 True로 예측한 샘플 중 실제 True인 비율  
  $$Precision = \frac{TP}{TP + FP}$$ 
* **재현율(Recall)**: 실제 True인 샘플 중 모델이 True로 예측한 비율  
  $$Recall = \frac{TP}{TP + FN}$$ 
* **F1-score**: 정밀도와 재현율의 조화평균으로, 클래스별 불균형 성과를 세부적으로 분석

* **멀티모달 확장**: 텍스트를 넘어 음성, 표정, 제스처 등 비언어적 요소를 결합한 감정 인식 모델로의 발전을 도모할 예정입니다.

## **7. 실험 재현 가이드 (Reproducibility)**

본 연구의 결과를 재현하기 위한 하드웨어 환경 및 소프트웨어 설정은 다음과 같습니다.

### **7.1 하드웨어 및 소프트웨어 환경**
* **OS**: Windows 11 Pro 또는 macOS Sonoma 
* **GPU**: Google Colab T4 GPU 환경
* **Language**: Python
* **핵심 라이브러리**: 
  * `torch`, `transformers` (v4.x 이상 권장)
  * `gluonnlp`, `mxnet` (KoBERT 데이터 처리를 위해 필수)
  * `pandas`, `numpy`, `tqdm`

### **7.2 토크나이저 및 데이터 처리 설정**
* **Tokenizer**: SKT Brain에서 제공하는 `KoBERTTokenizer`를 활용하였습니다 (`skt/kobert-base-v1`)
* **Dataset Class**: `BERTDataset` 클래스를 정의하여 토큰화, 패딩 및 어텐션 마스크 생성을 수행하였습니다.
* **입력 제한**:
  * `max_len`: 512 (긴 문장 처리를 위해 최대 길이로 설정)
  * `truncation`: True (최대 길이를 초과하는 문장은 절단)

### **7.3 하이퍼파라미터 (Hyper-parameters)**
실험에 사용된 최적의 하이퍼파라미터 설정값은 다음과 같습니다.

| 항목 | 설정값 | 
| :--- | :--- |
| **Batch Size** | 16 | 
| **Epochs** | 5 (Pre-training) / 3 (Validation) | 
| **Learning Rate** | $5 \times 10^{-5}$ | 
| **Warmup Ratio** | 0.1 |
| **Optimizer** | AdamW |
| **Loss Function** | CrossEntropyLoss |

### **7.4 실행 방법**
1. 저장소의 `kobert_for_class.ipynb` 파일을 Google Colab에서 엽니다.
2. 필수 라이브러리(`gluonnlp`, `mxnet`, `transformers` 등)를 설치합니다.
3. `label_mapping` 변수에 정의된 27개 감정 라벨이 데이터셋과 일치하는지 확인합니다.
4. 학습 루프를 실행하여 모델을 생성하고 성능을 검증합니다.
